- [首页重新设计：深色网格 → 天空蓝渐变 + 鼠标光斑] - 2026-07-14

  遇到的状况：原方案中深色 slate-950 基底 + 蓝色网格连接线 + 连接点的动效，用户认为"有问题"。实际想要的风格是类似 haoqi.design 的天空蓝渐变背景 + 鼠标牵引彩光光斑的生动效果。

  原因分析：
  - 原实现（MeshBackground.vue）是暗色系网格连接线 + 点阵漂浮，用户想要的是明亮天空蓝 + 柔光光斑跟随鼠标
  - 用户明确指出 haoqi.design 是开源可借鉴的，并且想要"天空蓝背景和光斑生动的色彩搭配"
  - 同时发现控制台报错 `Cannot read properties of undefined (reading 'r')`，原因是点阵负 y 坐标导致 COLORS[-1]

  考虑的方案：
  - 方案A：在当前 MeshBackground.vue 上增量修改（网格颜色变蓝 + 增加鼠标交互）— 但仍保留网格结构，不满足
  - 方案B：**全部重写 Canvas 背景**— 天空蓝渐变 + 多层鼠标跟随光斑 + 浮动粒子

  最终解决方案：采用方案 B，完全重写 MeshBackground.vue：
  - 天空蓝垂直渐变（#0a2a5e → #1a4f8e → #5b9bd5 → #a3cfe8 → #d4ecf7）
  - 4 层鼠标跟随彩光光斑（暖金/天空蓝/粉红/冰蓝），不同跟随速度产生层次拖尾感
  - 2 个巨大柔光层缓慢漂移，增添天空深度感
  - 80 个白色浮动粒子缓缓上升
  - 同时将所有组件配色从深色主题切换为毛玻璃浅色（bg-white/60 backdrop-blur-md）

  影响范围：MeshBackground.vue 完全重写；Home.vue、ProjectCard.vue、ContactBar.vue、AnimatedFish.vue、GuppyLM.vue 配色适配

- [详情页深色适配未完成] - 2026-07-14

  遇到的状况：首页改为深色主题（`slate-950`）后，点击项目卡片进入详情页（ProjectDetail.vue）背景仍为白色，产生闪白屏感。

  原因分析：全局配色方案从浅色（`bg-white` / `slate-50` 等）改为深色，但详情页 `min-h-screen bg-white` 及各元素的浅色类名未同步更新，详情页仍为白色背景。

  考虑的方案：
  - 方案A：逐一替换详情页所有颜色类名为深色系（slate-900, slate-800 等）
  - 方案B：使用 CSS 变量 + Tailwind dark mode 实现明暗主题切换（但当前无明暗切换需求，过度设计）

  最终解决方案：采用方案 A，将详情页的背景、导航条、图片区、标签、按钮等逐一替换为深色系。当前进度：
  - ✅ 页面背景、导航条、技术栈标签
  - ✅ 主图轮播容器、左右箭头
  - ✅ 缩略图导航（透明/蓝色边框，与深色融合）
  - ✅ 演示视频区（标题、容器边框、背景）
  - ✅ 项目介绍 & 贡献区（标题/正文/列表文字颜色）
  - ✅ 底部链接（分隔线、按钮边框、悬停状态）
  - ✅ 404 页面（背景、文字、按钮）
  - ✅ 操作按钮（GitHub/演示视频按钮的边框和文字色）

- [Canvas 网格背景动态流动] - 2026-07-14

  遇到的状况：用户希望借鉴 haoqi.design 的视觉氛围，但不能直接拷贝其原创设计。

  原因分析：haoqi.design 使用 Canvas WebGL 粒子效果 + 深色背景 + 荧光黄绿为主色调。

  考虑方案：
  - 方案A：直接下载/复制 haoqi.design 的背景（版权问题 ❌）
  - 方案B：自建 Canvas 网格流动动效，使用 Guppy 蓝色系（✅ 选择方案）

  最终解决方案：实现 MeshBackground.vue，绘制蓝色系网格点阵动画（spacing=80, AMPLITUDE=12），点之间有连接线 + 发光圆点，颜色沿垂直方向从 blue-500 渐变到 cyan-400，适配深色基底。
